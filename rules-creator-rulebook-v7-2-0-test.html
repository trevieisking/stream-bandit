Creator flow

1. Create

Prepare a title, description, public playable URL, poster, genres and tags.

2. Submit

Use Submit Video to create a pending submission row.

3. Review

Use Review Queue to approve, reject, request changes, or clear test rows.

4. Movie row

Only approved submissions move to the create-movie-row stage after preview.

This rulebook explains the flow. It does not change app data.

✅ Can do

🎬

Use public playable media

Use public Mux playback URLs, HLS links, or public video URLs that the player can open.

Allowed
🖼️

Use clean artwork

Use clear 16:9 poster or banner images, ideally 1920×1080, so cards and details pages look professional.

Allowed
🏷️

Add useful metadata

Add title, description, year, runtime, age rating, genres, tags and channel links where available.

Allowed
📡

Use creator tools

Use My Channel for a clean view, Channels for channel management, Submit for intake, and Review Queue for status.

Allowed

🚫 Do not

🔐

Do not paste private credentials

Do not paste private keys, private dashboard links, account tokens, passwords or personal login details into creator forms.

Never
⚠️

Do not skip review

Submissions should follow the pending to approved to preview to movie-row flow.

Never
🧱

Do not stack random patches

Complex pages should become clean full working versions instead of many small battle patches.

Avoid
🚦

Do not use this page for release actions

Release steps belong to readiness, smoke tests, backup checks and explicit approval.

Locked

⬆️ Submission rules

Title

Use a clear title and avoid placeholder names when ready for review.

Description

Give enough story/context so cards, details and search results make sense.

Video URL

Use a public playback link, not an editing dashboard or private account area.

Poster

Upload or provide a public artwork URL. Poster should stay with the submission.

Genres/tags

Use simple genres and tags for discovery and search.

Status

New submissions should start as pending where supported.

🧾 Review flow

Pending
