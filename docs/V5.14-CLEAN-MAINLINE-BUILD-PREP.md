# Stream Bandit V5.14 — Clean Mainline Build Prep

Date: 2026-05-07

## Purpose

This is the next safe step after V5.13 and V5.13.1 loader hygiene.

The goal is to prepare for a future clean mainline version of Stream Bandit where proven working fixes are folded into a cleaner app structure instead of being stacked as many late hotfix scripts.

## Current status

Current stable live continuation point: **V5.12.1 Stable**.

Hard restore checkpoint: **V5.11.8 Final Boss Controller**.

Accessibility/player comfort checkpoint: **V5.11.9 Player Sound Booster Restore**.

The main app file is still an older large core:

- `assets/stream-bandit-app.js`
- Internal app label currently still starts from `StreamBanditPrototype3.7`

Most newer fixes are currently layered on top as add-on scripts.

## Why not merge everything immediately?

A full merge into the main app is riskier than it looks because:

- The main app file is large.
- Many current fixes depend on render timing.
- Supabase, Mux, Cast & Crew, Details, Play, Watchlist, Likes and Sound Booster must not be broken.
- Previous issues came from multiple scripts fighting the same UI areas.

Therefore the safest order is:

1. Document and audit active scripts.
2. Identify isolated feature modules that can remain separate.
3. Identify shared UI patches that should later be merged or replaced.
4. Make one clean integration branch/build.
5. Test fully before changing the live loader.

## V5.14 build strategy

Recommended safest implementation:

### Step 1 — Create a clean UI controller

Create one new file:

`assets/stream-bandit-v5-14-ui-controller.js`

It should own only:

- version badge
- browser title
- Settings tabs
- Manager tabs
- safe manager layout cleanup
- duplicate old label cleanup

It should replace the need for:

- `stream-bandit-v5-11-8-final-boss-controller.js`
- `stream-bandit-v5-12-1-manager-layout-hotfix.js`

### Step 2 — Do not touch feature modules

Keep these loaded separately:

- Cast Manager
- Rating Calculator
- Sound Booster

### Step 3 — Use a test loader first

Before changing live `index.html`, create:

`test-v5-14.html`

This test loader can load:

- main app
- cast manager
- menu/settings logo if still needed
- rating calculator
- sound booster
- new V5.14 UI controller

Only after tests pass should `index.html` be updated.

## Summary

The next coding move should not be a direct rewrite of the giant app file.

Best safe path:

1. Create a V5.14 test loader.
2. Create one V5.14 UI controller to replace current UI patch stacking.
3. Keep feature modules separate.
4. Test test-v5-14.html first.
5. Only promote to live index.html after all tests pass.
