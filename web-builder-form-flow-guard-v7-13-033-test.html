V7.13.033 Form Flow Guard

Protect form flow from Web Builder corruption

This guard reads a Web Builder page and checks that form blocks still carry their preset fields and destination rules after Web Builder edits. It is read-only by default.

detects missing destinationsdetects missing fieldschecks presetsread-only checkerno schema change
Why this exists

A builder can accidentally save body blocks and strip the destination data. That would make a form look right but lose inbox/private/email routing.

Pass condition

Each form block keeps fields/questions, destination rules, and a matching preset entry in settings_json.

Next promotion

After this passes, we add the same guard into the real Form Designer save path before touching Full Preview.
