Overview

What Stream Bandit is

A streaming/video platform being rebuilt into safer standalone pages before final live promotion.

Current build style

Each page is test-only first, usually read-only first, then marked passed on the Master Map after testing.

Storage plan

Supabase handles app data and images. Mux/HLS/URL handles video playback paths.

Accessibility priority

Player comfort and louder audio remain important, especially because Trevor needs strong accessibility support.

Plan 3 rebuild

  • Build standalone pages one by one.
  • Keep live index.html untouched.
  • Mark each passed page on the Plan 3 Master Map.
  • Use the Tools Page style for real app-facing pages.
  • After all pages exist, build the final standalone app shell.
  • Create a release candidate before anything goes live.
Next after About: Accessibility standalone, then Supabase Library wrapper.

Safety rules

No live promotion

Nothing goes live until the release candidate passes and Trevor explicitly says promote live.

No dead links

Unfinished pages should become safe placeholders before final shell testing.

Read-only first

Admin, storage, review, and settings pages stay read-only first unless write actions are approved.

Artwork later

Missing 1920 × 1080 artwork is cleaned after logic and navigation pass.
