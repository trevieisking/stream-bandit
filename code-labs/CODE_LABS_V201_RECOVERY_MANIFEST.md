# Code Labs V201 Recovery Manifest

## Non-negotiable recovery rules

- One Code Labs V104 connector only.
- No separate connection, pairing, Sol, or V107 page.
- No manual Code Labs sign-in page dependency for live page control.
- Restore every protected workflow page and its working feature.
- File Lab remains the full current repository-file loader.
- Saved Files remains the current-file manager and gains the repository full-file puller.
- Buddy Canvas keeps the source/fixed full-file workspace but does not create repeating database saves.
- Every working page can update the same current saved-file row.
- Automatic page persistence overwrites that current row; it does not create new project, file, job, version, packet, or test rows.
- Checkpoints and explicit history snapshots remain deliberate version records only.
- GitHub writes remain connector-side, branch and pull request only.

## Acceptance gates

- The visible navigation no longer collapses to ten pages.
- Every protected page is reachable and keeps its page-specific loader or equivalent rewritten feature.
- Repo Desk, GitHub Writer, GitHub Tracker, Workflow Hub, Patch Desk, File Lab, and Buddy Canvas are usable.
- V104 can read, write, press buttons, read receipts, and undo on each working page.
