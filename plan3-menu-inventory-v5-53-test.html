Phase 3 — finish admin/tool pages

Then build or link: Supabase Manager, Supabase Test, Live Readiness, Supabase Migration, Mux Manager, Upload Plan, Local Storage, Storage Prep, Backup / Safety, Submit Video, Rules, Review Queue, Health Check, Test Checklist, Settings. Admin/write pages stay read-only first where possible.

Phase 4 — link passed pages together properly

Once a block passes, add proper navigation across the test pages. No old sidebar hash-routing. No broken links. No live index.html edits. Each page should know where to go next.

Phase 5 — build the new standalone app shell

Create app-shell-plan3-test.html. This becomes the replacement menu system. It must match the screenshot groups: WATCH, BROWSE, SUPABASE, MUX, STORAGE, ADMIN TOOLS. Each menu button links to a standalone file, not old in-app routing.

Phase 6 — placeholders for anything unfinished

No dead buttons. If a page is not ready, its menu button opens a safe placeholder saying: This page is not connected yet. Live app unchanged. Then we replace placeholders one by one.

Phase 7 — full shell smoke test

When the shell exists, test every group: WATCH links, BROWSE links, SUPABASE links, MUX links, STORAGE links, ADMIN TOOLS links, Details, Play, player controls, sound boost, fullscreen, no blank pages, and no jumping old admin tabs.

Phase 8 — artwork cleanup

Only after logic passes: fix missing 1920 × 1080 artwork, patch Supabase image fields, and keep image fixes separate from page logic.

Phase 9 — release candidate

When everything passes, create index-plan3-rc-test.html. That is the full test version before live. Still no live promotion.

Phase 10 — live promotion

Only after Trevor says all RC tests passed: backup current live version, promote once, test live immediately, and roll back if anything major breaks.
